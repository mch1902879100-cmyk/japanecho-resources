# Japanecho 云端音频资源与手机基础书籍设计

## 目标

将现有可播放专辑的原始音频完整备份到 GitHub Releases，同时为手机端提供体积更小、可选择安装的播放资源包。手机 APK 只保留少量基础书籍，并在界面中清楚区分“内置”和“已下载”内容。

本设计采用已经确认的双层方案：

- 原始备份层：按原格式保存现有全部音频文件，用于长期备份和恢复，不在手机资源中心显示。
- 手机播放层：从原始音频生成适合手机播放的压缩音频资源包，在手机资源中心显示并可安装。

## 范围

原始备份层覆盖 `E:\合成测试\03_输出文件` 中现有的全部音频文件。当前扫描快照约为 14,755 个音频文件、3.43 GB，包含正式专辑、实验目录和其他历史输出。

手机播放层只处理 `E:\合成测试\03_输出文件\albums` 中与书架目录对应、能够正常播放的 83 个专辑，当前约为 12,194 个音频文件、3.13 GB。临时粘贴文本、重复实验目录和未进入书架目录的散落音频只进入原始备份，不进入手机书架。

- 宫泽贤治《よだかの星》：`classic-miyazawa-07-7`

## 资源分层

### 原始音频备份

- 全量扫描输出目录，正式专辑和未整理历史输出都必须进入备份清单。
- 按来源目录分卷打包，单个 Release 文件必须小于 2 GiB。
- 保留原始目录结构、`manifest.json`、`segments.json`、封面和全部原始音频。
- 发布到 GitHub Releases，但在手机资源目录中标记为不可见。
- 用于灾难恢复、重新转码和未来制作其他平台资源包。

### 手机播放包

- 从原始音频生成 96 kbps 单声道 MP3，兼顾语音清晰度和手机存储占用。
- 保留播放器需要的文本、翻译、分段、封面和目录信息。
- 按合集拆分资源包；较大的合集继续分卷，避免手机下载和解压时占用过多内存。
- 每个播放包以 200 MB 为目标，250 MB 为硬上限；超出时自动拆分。
- 手机资源中心只展示此类播放包。

## 资源目录格式

`resources.json` 中每个资源增加以下语义字段：

```json
{
  "kind": "playback",
  "mobileVisible": true,
  "includesAudio": true,
  "albumIds": ["album-id-1", "album-id-2"]
}
```

字段规则：

- `kind: "playback"`：手机可安装播放包。
- `kind: "backup"`：原始备份包。
- `mobileVisible`：是否在手机资源中心显示。
- `includesAudio`：资源包是否包含可播放音频。
- `albumIds`：资源包内包含的作品 ID，用于判断内置、已下载和重复内容。

现有版本、大小、SHA-256、下载地址和最低应用版本字段继续保留。

## 手机端展示规则

采用已确认的 A + C 方案：

- APK 自带作品在封面右上角显示绿色 `内置` 角标。
- 从资源中心安装的作品显示紫色 `已下载` 角标。
- 普通书架只显示当前可播放的作品。
- 未下载的云端作品不在普通书架显示，只在资源中心中显示。
- 资源中心显示资源包大小、作品数量、内置数量和已下载状态。

当同一个作品同时存在于 APK 和下载资源包中时，优先使用 APK 内置版本，避免重复显示和重复占用。卸载资源包不会影响内置作品。

## 数据流

1. 扫描完整输出目录，生成全量原始音频备份清单。
2. 单独扫描书架目录和专辑目录，生成手机播放专辑清单。
3. 检查每个手机播放专辑的文本、翻译、音频路径和可播放状态。
4. 将全部原始音频按备份分卷打包并生成校验值。
5. 将正式专辑的原始音频转换为手机播放 MP3，按合集生成播放资源包。
6. 生成统一 `resources.json`，其中备份包对手机隐藏。
7. 发布资源目录和所有资源包到 GitHub Releases。
8. 手机端读取资源目录，只展示可见播放包。
9. 安装完成且校验成功后，作品进入普通书架并显示 `已下载`。

## 安装与合并规则

- 下载完成后先校验文件大小和 SHA-256，再开始安装。
- 解压和目录写入全部成功后，才更新“已安装资源”记录。
- 安装中断或校验失败时，不把半成品加入书架。
- 资源包内作品 ID 与内置作品 ID 重复时，内置版本优先。
- 资源包升级时，新版本安装成功后再移除旧版本。
- 删除资源包只删除该资源包安装的文件，不影响 APK 内置文件和其他资源包。

## 错误处理

- 发布前拒绝缺少清单、分段文件或音频文件的专辑。
- 发布前拒绝重复作品 ID、重复资源包 ID 和超过大小上限的分卷。
- 下载失败、校验失败和安装失败均提供可重试状态。
- GitHub Releases 中的原始备份包即使手机播放包生成失败也保持可恢复。
- 资源目录读取失败时，手机仍可正常使用内置和已安装内容。

## 验证标准

- 全部 14,755 个现有输出音频都能在原始备份清单中找到；若发布前文件数量变化，以发布时重新扫描的清单为准。
- 83 个正式书架专辑都能在手机播放清单中找到。
- 原始音频备份包解压后可恢复原始输出目录和专辑目录。
- 手机播放包中的每个作品至少抽查一段原文和一段中文音频可播放。
- 所有发布文件的大小和 SHA-256 与 `resources.json` 一致。
- APK 中只包含批准的 7 个基础作品。
- 手机普通书架不显示未下载云端作品。
- 内置作品显示绿色 `内置`，安装资源显示紫色 `已下载`。
- 同一作品同时出现在内置与资源包时只显示一次，并使用内置版本。
- 资源包安装、升级、卸载和失败重试均通过测试。

## 非目标

- 本次不更改 TTS 生成逻辑、翻译模型、播放器流程或书架层级。
- 本次不把原始音频备份包直接作为手机资源中心下载内容。
- 本次不实现付费、账户同步或私有云权限控制。
